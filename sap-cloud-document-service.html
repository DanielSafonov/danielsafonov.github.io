SAP Cloud Document Service

17 May 2019 | sap spring java

Работа с SAP Cloud Document Service на Java.

SAP Cloud Platform Document Service - ECM-система от SAP.

ECM - Enterprise content management (cистема управления корпоративным контентом) - управление цифровыми документами и другими типами контента, а также их хранение, обработка и доставка в рамках организации. Управляемая информация (контент) предполагает слабую структурированность: это могут быть файлы различных форматов, электронные документы с различными наборами полей.

Общее файловое пространство внутри Document Service разделяется на репозитории, дальнейшее взаимодествие идет именно с конкретным репозиторием. Document Service использует OASIS стандарт спецификации CMIS.

CMIS - Content Management Interoperability Services (сервисы взаимодействия при управлении контентом) - предлагаемый ведущими производителями пакет стандартов, состоящий из набора веб-сервисов для совместного использования информации, хранимой в несвязанных между собой хранилищах контента.

Официальная документация по взаимодействию с SAP Cloud Document Service содержит пример простого приложения на Java.

Apache Chemistry - OpenSource библиотека, реализующая стандарт CMIS для взаимодействия с ECM-системой.

  • Apache Chemistry JavaDoc
  • Apache Chemistry Code Samples


Так как информации в сети практически нет, изучал код OpenSource проекта Eclipse Dirijible, который реализует взаимодействие с SAP Cloud Document Service.


Стоит отменить, что у меня достаточно много времени ушло на выставление правильных зависимостей для Spring Boot приложения. В результате получилось следующее:

<properties>
   <neo.sdk.version>3.78.15</neo.sdk.version>
   <apacheChemistry.version>1.1.0</apacheChemistry.version>
   ...
</properties>

...

<dependency>
    <groupId>com.sap.cloud</groupId>
    <artifactId>neo-java-web-api</artifactId>
    <version>${neo.sdk.version}</version>
    <scope>provided</scope>
</dependency>

<dependency>
    <groupId>org.apache.chemistry.opencmis</groupId>
    <artifactId>chemistry-opencmis-client-impl</artifactId>
    <version>${apacheChemistry.version}</version>
    <scope>provided</scope>
    <exclusions>
        <exclusion>
	        <groupId>org.slf4j</groupId>
	        <artifactId>slf4j-api</artifactId>
     	</exclusion>
        <exclusion>
            <groupId>org.apache.chemistry.opencmis</groupId>
            <artifactId>chemistry-opencmis-client-api</artifactId>
        </exclusion>
        <exclusion>
            <groupId>org.apache.chemistry.opencmis</groupId>
            <artifactId>chemistry-opencmis-commons-api</artifactId>
        </exclusion>
    </exclusions>
</dependency>

Помимо этого, web.xml должен содержать определение ресурса Ecm-сервиса:

<resource-ref>
	<res-ref-name>EcmService</res-ref-name>
   	<res-type>com.sap.ecm.api.EcmService</res-type>
</resource-ref>

Примеры кода

Создать новый репозиторий:

//Параметры создаваемого репозитория
RepositoryOptions repositoryOptions = new RepositoryOptions();
repositoryOptions.setUniqueName(repositoryDTO.getRepositoryName());
repositoryOptions.setRepositoryKey(repositoryDTO.getRepositoryKey());
repositoryOptions.setVisibility(RepositoryOptions.Visibility.PROTECTED);
repositoryOptions.setDisplayName(repositoryDTO.getRepositoryName());
if (repositoryDTO.getRepositoryDescription() != null) {
	repositoryOptions.setDescription(repositoryDTO.getRepositoryDescription());
}
repositoryOptions.setVirusScannerEnabled(repositoryDTO.getIsVirusScannerEnabled());

//Валидация параметров репозитория
if(repositoryOptions.validate() != null){
	//Параметры не прошли валидацию
	String error = repositoryOptions.validate();
	...
}

...

//Создание репозитория
try{
	ecmService.createRepository(repositoryOptions);
} catch (RepositoryAlreadyExistsException e) {
	//Репозиторий с таким именем уже существует
	...
} catch (ServiceException e) {
	//Ошибка ECM-сервиса
	...
} catch (Exception e) {
	//Ошибка общего характера
	...
}

Важно: в моем случае Document Service блокировал создание репозитория с любым типом Visibility, кроме PROTECTED.

Соединиться с репозиторием:

try{
	//Открыть соединение
	session = ecmService.connect(repositoryName, repositoryKey);
} catch (CmisObjectNotFoundException e) {
	//Репозиторий не найден
	...
} catch (ServiceException e) {
	//Ошибка ECM-сервиса
	...
} catch (Exception e) {
	//Ошибка общего характера
	...
}

Удалить репозиторий:

try{
	ecmService.deleteRepository(repositoryName, repositoryKey);
} catch (RepositoryNotEmptyException e) {
	//Репозиторий содержит какие-то файлы

	if (repositoryDTO.getIsForceDelete() == true) {
		//Принудительное удаление репозитория
		ecmService.forceDeleteRepository(repositoryName, repositoryKey);
		return true;
	} else {
		...

All the Eclipse Dirigible project’s source code and sample applications are licensed under Eclipse Public License - v 1.0 and maintained at GitHub.

Eclipse Dirigible умеет работать с ECM-системами по стандарту CMIS (в том числе и с SAP Cloud Document Service), написан на Java и использует библиотеку Apache Chemistry. Я изучил его исходники для понимания Best Practice по работе с ECM-системами и библиотекой Apache Chemistry.
