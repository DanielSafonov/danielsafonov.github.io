.gitignore

21 May 2019 | git

Игнорирование файлов в git.

Зачастую, у вас имеется группа файлов, которые вы не только не хотите автоматически добавлять в репозиторий, но и видеть в списках неотслеживаемых. К таким файлам обычно относятся автоматически генерируемые файлы (различные логи, результаты сборки программ и т.п.). В таком случае, вы можете создать файл .gitignore с перечислением шаблонов соответствующих таким файлам.

Основы Git - Запись изменений в репозиторий

Файл .gitignore создается в корневом каталоге рабочей директории.

$ touch .gitignore #Создать файл
$ nano .gitignore #Перейти к редактированию файла



#Пример файла .gitignore
/.settings/
/.classpath
*.iml
.idea/
/target/

*.log
*.gz

.DS_Store



#Помимо этого, сущестсует глобальный .gitignore
$ git config --global core.excludesfile ~/.gitignore_global

Правила синтаксиса:

  • Пустые строки игнорируются
  • Строки с # - комментарии
  • target/ - директория
  • target - файл или директория
  • /target - файл или директория в корневом каталоге
  • .class - все файлы с расширением .class
  • /*.class - все файлы с расширением .class из корневого каталога
  • !target - не игнорировать файл или директорию
  • Символ \ используется для экранирования спецсимволов
  • ** используются для указания любого количества поддиректорий
  • * - любое количество символов

Если файл уже был проиндексирован и после внесен в .gitignore, игнорирование не применится. Необходимо вручную исключить его из индекса, после чего правила игнорирования примут силу:

$ git rm FILENAME
$ git rm --cached FILENAME
