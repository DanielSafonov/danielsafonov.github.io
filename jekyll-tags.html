Jekyll Tags

01 May 2019 | jekyll web

Jekyll не имеет встроенной поддержки системы тегов, но ее можно легко добавить.

Метаданные поста - каждый пост может содержать неограниченное количество тэгов.

---
layout: post 
title: Jekyll Tags
tags: [jekyll, web]
---


Топ 5 тэгов: листнинг выводит 5 тэгов с наибольшим количеством постов.

<!--Сортировка тэгов по количеству постов-->
{% capture tags %}
  {% for tag in site.tags %}
    {{ tag[1].size | plus: 1000 }}#{{ tag[0] }}#{{ tag[1].size }}
  {% endfor %}
{% endcapture %}
{% assign sortedtags = tags | split:' ' | sort %}
<!-- Смещение для вывода 5 тэгов в цикле -->
{% assign offset = sortedtags.size | minus:5 %}

<!-- Вывод топ-5 тэгов -->
<div class="toptags">
  {% for tag in sortedtags reversed offset:offset %}
    {% assign tagitems = tag | split: '#' %}
    <a href="#{{ tagitems[1] | slugify }}" class="badge tag">{{ tagitems[1] }}</a>
  {% endfor %}  
</div>


Страница Sitemap: выводит отсортированное по количеству постов облако тэгов, а также список постов (название и дата) по каждому из тэгов последовательно.

<!--Сортировка тэгов по количеству постов-->
{% capture tags %}
  {% for tag in site.tags %}
    {{ tag[1].size | plus: 1000 }}#{{ tag[0] }}#{{ tag[1].size }}
  {% endfor %}
{% endcapture %}

{% assign sortedtags = tags | split:' ' | sort %}

  <!--Вывод отсортированной стороки с тэгами и количеством постов по каждому тэгу-->
  <div class="tags-expo">
    {% for tag in sortedtags reversed %}
    {% assign tagitems = tag | split: '#' %}
      <a href="#{{ tagitems[1] | slugify }}" class="badge tag">{{ tagitems[1] }} ({{ tagitems[2] }})</a>
    {% endfor %}  
  <hr/>

  <!-- Вывод отсортированного списка тэгов и ссылки на посты по каждому -->
  <div class="tags-expo-section">
    {% for tag in site.tags %}
    <h2 id="{{ tag[0] | slugify }}">{{ tag[0] }}</h2>
    <ul class="tags-expo-posts">
      {% for post in tag[1] %}
        <a class="post-title" href="{{ site.baseurl }}{{ post.url }}">
          <li> {{ post.title }} <small class="post-date">{{ post.date | date_to_string }}</small> </li>
        </a>
      {% endfor %}
    </ul>
    {% endfor %}
  </div>
</div>
