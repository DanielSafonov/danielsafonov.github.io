13 Jun 2019 | spring java qa

‌‌

Некоторые виды тестирования ПО:

  • Модульное тестирование (unit testing) - тестирование минимально возможных частей приложения по отдельности
  • Интеграционное тестирование (integration testing) - тестирование взаимодействия компонентов приложения
  • Нагрузочное тестирование (load testing) - тестирование приложения при стандартных нагрузках для определения максимального пика, при котором система работает стабильно - эмуляции работы нескольких пользователей одновременно
  • Стресс тестирование (stress testing) - тестирование приложения при нестандартных нагрузках для определения максимального пика для выявления максимального пика нагрузки и условий, при которых система выходит из строя
  • Системное тестирование (system testing) - тестирование работы всего приложения

Unit tests

  • тестирование минимально возможных функциональных частей приложения
  • тесты не должны загружать Spring-контекст (@RunWith(SpringRunner.class))
  • тестируются только public методы; внутренний код тестируется за счет изменения входных данных
  • каждый класс имеет свой тест
  • внешние ресурсы не должны использоваться: недопустимо подключение к БД, почтовым/веб-службам и тд, поэтому юнит-тесты репозиториев недопустимы - для таких задач необходимо писать интеграционные тесты
  • для получения данных из внешних ресурсов или взаимодействия с другими классами, требуемыго тестируемой логике, должны создаваться фиктивные объекты (моки)
