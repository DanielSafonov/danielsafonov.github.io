Swagger - UI, Spring Boot

12 Apr 2020 | java spring api

Современные веб-приложения, как правило, состоят из двух отдельных частей - frontend и backend. В качестве фронтенда может выступать как страница в браузере, так и мобильное или десктопное приложение. Для взаимодействия c фронендом бэкенд выставляет один или множество API.

Соответственно, API должен иметь документацию, чтобы фронтенд-разработчики могли с ним взаимодействовать. Более того, документация (спецификация) API должна быть актуальна в каждый момент времени.

Именно такие задачи решают фреймворки для спецификации API, одним из которых является Swagger.


Swagger takes the manual work out of API documentation, with a range of solutions for generating, visualizing, and maintaining API docs.

Swagger Open Source Tools

  • Swagger Codegen: Generate server stubs and client SDKs from OpenAPI Specification definitions;
  • Swagger Editor: API editor for designing APIs with the OpenAPI Specification;
  • Swagger UI: Visualize OpenAPI Specification definitions in an interactive UI.


OpenAPI Specification (OAS), ранее известный как Swagger Specification (OAS 2.0), это стандарт для описания RESTful API. Актуальная версия - OAS 3.0.

OpenAPI файл позволяет полность описать API, включая

  • Доступные эндпоинты и операции на каждом из них (GET /users, POST /users)
  • Параметры запроса и ответа на каждый endpoint
  • Методы аутентификации
  • Контактную информацию, лицензию и прочую информацию об API

Swagger actial version - 2.1.2 (OAS 3.0)
